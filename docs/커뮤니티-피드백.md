# 커뮤니티 피드백 시스템

## 개요

프로젝트 커뮤니티에서 사용자가 버그 리포트, 기능 요청, 개선 제안, 질문 등의 피드백을 작성하고 관리하는 시스템입니다. 모든 사용자가 피드백을 작성할 수 있으며, 프로젝트 개발팀은 피드백을 관리하고 공식 답변을 작성할 수 있습니다.

### 주요 기능

- **피드백 작성/수정/삭제**: 버그 리포트, 기능 요청, 개선 제안, 질문 작성
- **피드백 관리**: 상태 변경, 우선순위 설정, 담당자 지정
- **공식 답변**: 개발팀이 피드백에 대한 개발자 답변 작성
- **투표 기능**: 사용자가 피드백에 투표하여 인기 피드백 확인
- **이미지 첨부**: 최대 3개 이미지 첨부 가능
- **댓글 시스템**: 피드백별 댓글 작성/수정/삭제 (최대 3단계 깊이)
- **필터링**: 타입별, 상태별 필터링
- **정렬**: 투표수 순, 최신순 정렬
- **페이지네이션**: 30개씩 페이지네이션 지원
- **상세 페이지**: 피드백 상세 정보 조회 및 관리

## 데이터베이스 스키마

### 1. 피드백 테이블 (`odd.tbl_feedbacks`)

피드백을 저장하는 테이블입니다. `tbl_posts`와 1:1 관계입니다.

#### 스키마

```sql
CREATE TABLE odd.tbl_feedbacks (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    post_id uuid NOT NULL REFERENCES odd.tbl_posts(id) ON DELETE CASCADE,
    
    -- 피드백 정보
    title text NOT NULL,  -- 제목
    feedback_type text NOT NULL CHECK (feedback_type IN ('bug', 'feature', 'improvement', 'question')),  -- 피드백 타입
    status text NOT NULL DEFAULT 'open' CHECK (status IN ('open', 'in_progress', 'resolved', 'closed')),  -- 상태
    priority text CHECK (priority IN ('low', 'medium', 'high', 'critical')),  -- 우선순위
    
    -- 담당자 및 답변
    assignee_id bigint REFERENCES odd.tbl_users(id) ON DELETE SET NULL,  -- 담당자
    developer_response text,  -- 개발팀 공식 답변
    
    -- 상태
    is_pinned boolean DEFAULT false NOT NULL,
    
    -- 통계 (비정규화)
    votes_count integer DEFAULT 0 NOT NULL,
    
    -- 타임스탬프
    created_at timestamptz DEFAULT now() NOT NULL,
    updated_at timestamptz DEFAULT now() NOT NULL,
    
    -- 하나의 포스트는 하나의 피드백만 가질 수 있음
    UNIQUE(post_id)
);
```

#### 주요 컬럼 설명

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `id` | uuid | ✅ | 고유 ID (자동 생성) |
| `post_id` | uuid | ✅ | 포스트 ID (`odd.tbl_posts.id` 참조) |
| `title` | text | ✅ | 제목 |
| `feedback_type` | text | ✅ | 피드백 타입: `bug`, `feature`, `improvement`, `question` |
| `status` | text | ✅ | 상태: `open`, `in_progress`, `resolved`, `closed` (기본값: `open`) |
| `priority` | text | - | 우선순위: `low`, `medium`, `high`, `critical` |
| `assignee_id` | bigint | - | 담당자 ID (`odd.tbl_users.id` 참조) |
| `developer_response` | text | - | 개발팀 공식 답변 |
| `is_pinned` | boolean | ✅ | 상단 고정 여부 |
| `votes_count` | integer | ✅ | 투표 수 (비정규화, 트리거로 자동 동기화) |
| `created_at` | timestamptz | ✅ | 생성일시 |
| `updated_at` | timestamptz | ✅ | 수정일시 (트리거로 자동 업데이트) |

#### 인덱스

| 인덱스명 | 컬럼 | 용도 |
|---------|------|------|
| `idx_tbl_feedbacks_post_id` | `post_id` | 포스트 ID로 조회 |
| `idx_tbl_feedbacks_type` | `feedback_type` | 피드백 타입별 조회 |
| `idx_tbl_feedbacks_status` | `status` | 상태별 조회 |
| `idx_tbl_feedbacks_priority` | `priority` (WHERE `priority IS NOT NULL`) | 우선순위별 조회 |
| `idx_tbl_feedbacks_assignee` | `assignee_id` (WHERE `assignee_id IS NOT NULL`) | 담당자별 조회 |
| `idx_tbl_feedbacks_pinned` | `is_pinned` (WHERE `is_pinned = true`) | 고정된 피드백 조회 |
| `idx_tbl_feedbacks_votes` | `votes_count DESC` | 투표수 정렬 (인기순) |

### 2. 피드백 투표 테이블 (`odd.tbl_feedback_votes`)

피드백 투표를 저장하는 테이블입니다. 한 사용자는 한 피드백에 하나의 투표만 가능합니다.

#### 스키마

```sql
CREATE TABLE odd.tbl_feedback_votes (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    feedback_id uuid NOT NULL REFERENCES odd.tbl_feedbacks(id) ON DELETE CASCADE,
    user_id bigint NOT NULL REFERENCES odd.tbl_users(id) ON DELETE CASCADE,
    
    created_at timestamptz DEFAULT now() NOT NULL,
    
    -- 한 사용자는 한 피드백에 하나의 투표만 가능
    UNIQUE(feedback_id, user_id)
);
```

#### 주요 컬럼 설명

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `id` | uuid | ✅ | 고유 ID (자동 생성) |
| `feedback_id` | uuid | ✅ | 피드백 ID (`odd.tbl_feedbacks.id` 참조) |
| `user_id` | bigint | ✅ | 투표한 사용자 ID (`odd.tbl_users.id` 참조) |
| `created_at` | timestamptz | ✅ | 생성일시 |

#### 인덱스

| 인덱스명 | 컬럼 | 용도 |
|---------|------|------|
| `idx_tbl_feedback_votes_feedback_id` | `feedback_id` | 피드백별 투표 조회 |
| `idx_tbl_feedback_votes_user_id` | `user_id` | 사용자별 투표 조회 |

#### 트리거

- **`trigger_increment_feedback_votes_count`**: 피드백 투표 추가 시 피드백의 `votes_count` 증가
- **`trigger_decrement_feedback_votes_count`**: 피드백 투표 삭제 시 피드백의 `votes_count` 감소

## RPC 함수

### 1. 피드백 목록 조회 (`odd.v1_fetch_feedbacks`)

피드백 목록을 조회하는 함수입니다. 필터링, 정렬, 페이지네이션을 지원합니다.

#### 함수 시그니처

```sql
CREATE OR REPLACE FUNCTION odd.v1_fetch_feedbacks(
    p_project_id uuid,
    p_feedback_type text DEFAULT NULL,
    p_status text DEFAULT NULL,
    p_limit integer DEFAULT 30,
    p_offset integer DEFAULT 0
)
RETURNS TABLE (
    -- 포스트 기본 정보
    id uuid,
    author_id bigint,
    type text,
    content text,
    images jsonb,
    likes_count integer,
    comments_count integer,
    is_pinned boolean,
    created_at timestamptz,
    updated_at timestamptz,
    -- 작성자 정보
    author_username text,
    author_display_name text,
    author_avatar_url text,
    -- 현재 사용자 인터랙션 상태
    is_liked boolean,
    -- 피드백 정보
    post_id uuid,
    feedback_id uuid,
    title text,
    feedback_type text,
    status text,
    priority text,
    assignee_id bigint,
    developer_response text,
    votes_count integer,
    is_voted boolean
)
```

#### 매개변수

| 매개변수 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| `p_project_id` | uuid | ✅ | 프로젝트 ID |
| `p_feedback_type` | text | - | 피드백 타입 필터: `bug`, `feature`, `improvement`, `question` (NULL이면 모든 타입) |
| `p_status` | text | - | 상태 필터: `open`, `in_progress`, `resolved`, `closed` (NULL이면 모든 상태) |
| `p_limit` | integer | - | 조회 개수 제한 (기본값: 30, 최대: 100) |
| `p_offset` | integer | - | 페이지네이션 오프셋 (기본값: 0) |

#### 반환 데이터 구조

```typescript
interface Feedback {
  // 포스트 기본 정보
  id: string; // post_id
  feedbackId: string; // 실제 feedback_id
  author_id: number;
  type: string;
  content: string;
  images: string[];
  likes_count: number;
  comments_count: number;
  is_pinned: boolean;
  created_at: string;
  updated_at: string;
  
  // 작성자 정보
  author_username: string;
  author_display_name: string;
  author_avatar_url?: string;
  
  // 현재 사용자 인터랙션 상태
  is_liked: boolean;
  
  // 피드백 정보
  title: string;
  feedback_type: 'bug' | 'feature' | 'improvement' | 'question';
  status: 'open' | 'in_progress' | 'resolved' | 'closed';
  priority?: 'low' | 'medium' | 'high' | 'critical';
  assignee_id?: number;
  developer_response?: string;
  votes_count: number;
  is_voted: boolean;
}
```

#### 사용 예시

```typescript
import { supabase } from "@/shared/lib/supabase";

// 전체 피드백 조회
const { data, error } = await supabase
  .schema('odd')
  .rpc('v1_fetch_feedbacks', {
    p_project_id: 'project-uuid-here',
    p_limit: 30,
    p_offset: 0
  });

// 기능 요청만 조회
const { data, error } = await supabase
  .schema('odd')
  .rpc('v1_fetch_feedbacks', {
    p_project_id: 'project-uuid-here',
    p_feedback_type: 'feature',
    p_limit: 30,
    p_offset: 0
  });

// 진행 중인 피드백만 조회
const { data, error } = await supabase
  .schema('odd')
  .rpc('v1_fetch_feedbacks', {
    p_project_id: 'project-uuid-here',
    p_status: 'in_progress',
    p_limit: 30,
    p_offset: 0
  });
```

#### 정렬 규칙

1. 고정된 피드백을 먼저 표시 (`is_pinned DESC`)
2. 투표수 높은 순 (인기순) (`votes_count DESC`)
3. 최신순 정렬 (`created_at DESC`)

### 3. 피드백 생성 (`odd.v1_create_feedback`)

```sql

    p_project_id uuid,
    p_feedback_type text,  -- 'bug' | 'feature' | 'improvement' | 'question'

    p_content text,
    p_images jsonb DEFAULT '[]'::jsonb
)
RETURNS uuid
```

| 매개변수 | 타입 | 필수 | 설명 |

| `p_content` | text | ✅ | 내용 |

### 4. 피드백 수정 (`odd.v1_update_feedback`)

피드백을 수정하는 함수입니다. 작성자는 제목, 내용, 이미지만 수정 가능하고, 프로젝트 멤버는 모든 필드를 수정할 수 있습니다.

#### 함수 시그니처

```sql
CREATE OR REPLACE FUNCTION odd.v1_update_feedback(
    p_post_id uuid,
    p_title text DEFAULT NULL,
    p_content text DEFAULT NULL,
    p_images jsonb DEFAULT NULL,
    p_feedback_type text DEFAULT NULL,
    p_status text DEFAULT NULL,
    p_priority text DEFAULT NULL,
    p_assignee_id bigint DEFAULT NULL,
    p_developer_response text DEFAULT NULL,
    p_is_pinned boolean DEFAULT NULL
)
RETURNS boolean
```

#### 매개변수

| 매개변수 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| `p_post_id` | uuid | ✅ | 포스트 ID |
| `p_title` | text | - | 제목 (NULL이면 변경하지 않음) |
| `p_content` | text | - | 내용 (NULL이면 변경하지 않음) |
| `p_images` | jsonb | - | 이미지 URL 배열 (NULL이면 변경하지 않음) |
| `p_feedback_type` | text | - | 피드백 타입 (NULL이면 변경하지 않음) |
| `p_status` | text | - | 상태 (NULL이면 변경하지 않음) |
| `p_priority` | text | - | 우선순위 (NULL이면 변경하지 않음) |
| `p_assignee_id` | bigint | - | 담당자 ID (NULL이면 변경하지 않음) |
| `p_developer_response` | text | - | 공식 답변 (NULL이면 변경하지 않음) |
| `p_is_pinned` | boolean | - | 상단 고정 여부 (NULL이면 변경하지 않음) |

#### 권한

- 작성자: 제목, 내용, 이미지만 수정 가능
- 프로젝트 멤버: 모든 필드 수정 가능

#### 사용 예시

```typescript
import { supabase } from "@/shared/lib/supabase";

// 작성자가 피드백 수정
const { data, error } = await supabase
  .schema('odd')
  .rpc('v1_update_feedback', {
    p_post_id: 'post-uuid-here',
    p_title: '수정된 제목',
    p_content: '수정된 내용',
    p_images: ['url1', 'url2', 'url3']
  });

// 프로젝트 멤버가 상태 변경
const { data, error } = await supabase
  .schema('odd')
  .rpc('v1_update_feedback', {
    p_post_id: 'post-uuid-here',
    p_status: 'in_progress',
    p_priority: 'high',
    p_assignee_id: 123,
    p_developer_response: '검토 중입니다.'
  });
```

## 프론트엔드 사용 방법

### 1. 피드백 목록 조회

```typescript
import { supabase } from "@/shared/lib/supabase";

async function fetchFeedbacks(projectId: string, options?: {
  feedbackType?: 'bug' | 'feature' | 'improvement' | 'question';
  status?: 'open' | 'in_progress' | 'resolved' | 'closed';
  limit?: number;
  offset?: number;
}) {
  const { data, error } = await supabase
    .schema('odd')
    .rpc('v1_fetch_feedbacks', {
      p_project_id: projectId,
      p_feedback_type: options?.feedbackType || null,
      p_status: options?.status || null,
      p_limit: options?.limit || 30,
      p_offset: options?.offset || 0
    });

  if (error) {
    console.error('피드백 조회 실패:', error);
    return { error };
  }

  return { data };
}
```

### 2. 피드백 상세 조회

```typescript
import { supabase } from "@/shared/lib/supabase";

async function fetchFeedbackDetail(postId: string) {
  const { data, error } = await supabase
    .schema('odd')
    .rpc('v1_fetch_feedback_detail', {
      p_post_id: postId
    });

  if (error) {
    console.error('피드백 상세 조회 실패:', error);
    return { error };
  }

  if (!data || data.length === 0) {
    return { error: new Error('피드백을 찾을 수 없습니다') };
  }

  return { data: data[0] };
}
```

### 3. 피드백 생성

```typescript
import { supabase } from "@/shared/lib/supabase";
import { uploadPostImages } from "@/shared/lib/storage";

async function createFeedback(feedbackData: {
  projectId: string;
  feedbackType: 'bug' | 'feature' | 'improvement' | 'question';
  title: string;
  content: string;
  images?: File[];
}) {
  // 이미지 업로드
  let imagePaths: string[] = [];
  if (feedbackData.images && feedbackData.images.length > 0) {
    const { data: { user } } = await supabase.auth.getUser();
    if (!user) {
      return { error: new Error('로그인이 필요합니다') };
    }

    const tempPostId = `temp-${Date.now()}`;
    const { paths, error: uploadError } = await uploadPostImages(
      feedbackData.images,
      user.id,
      tempPostId
    );

    if (uploadError) {
      return { error: uploadError };
    }

    imagePaths = paths;
  }

  // 피드백 생성
  const { data, error } = await supabase
    .schema('odd')
    .rpc('v1_create_feedback', {
      p_project_id: feedbackData.projectId,
      p_feedback_type: feedbackData.feedbackType,
      p_title: feedbackData.title,
      p_content: feedbackData.content,
      p_images: imagePaths
    });

  if (error) {
    console.error('피드백 생성 실패:', error);
    return { error };
  }

  return { data };
}
```

## 보안 및 권한 (RLS)

### 피드백 테이블 (`odd.tbl_feedbacks`)

#### 읽기 권한
- 모든 사용자가 읽을 수 있음
- 정책: `"Anyone can read feedbacks"` (SELECT, TO public)

#### 쓰기 권한

**피드백 생성 (INSERT):**
- 인증된 사용자만 피드백 생성 가능
- 정책: `"Authenticated users can create feedbacks"` (INSERT, TO authenticated)

**피드백 수정 (UPDATE):**
- 작성자는 제목, 내용, 이미지만 수정 가능
- 프로젝트 멤버는 모든 필드 수정 가능
- 정책: `"Users can update own feedbacks"` (UPDATE, TO authenticated)
- 권한 체크는 함수에서 수행

### 피드백 투표 테이블 (`odd.tbl_feedback_votes`)

#### 읽기 권한
- 모든 사용자가 읽을 수 있음
- 정책: `"Anyone can read feedback votes"` (SELECT, TO public)

#### 쓰기 권한
- 인증된 사용자만 투표 가능
- 자신의 투표만 삭제 가능
- 정책: `"Authenticated users can create feedback votes"` (INSERT, TO authenticated)
- 정책: `"Users can delete own feedback votes"` (DELETE, TO authenticated)

## 에러 처리

### 일반적인 에러

#### 인증 에러
- **에러 메시지**: `"인증이 필요합니다"`
- **원인**: 로그인하지 않은 사용자가 인증이 필요한 작업을 시도
- **해결**: 로그인 페이지로 리다이렉트

#### 권한 에러
- **에러 메시지**: `"피드백을 찾을 수 없거나 수정 권한이 없습니다"`
- **원인**: 작성자가 아닌 사용자가 수정 시도 또는 프로젝트 멤버가 아닌 사용자가 관리 기능 시도
- **해결**: 권한 확인 후 작업 수행

### 피드백 관련 에러

#### 피드백 타입 에러
- **에러 메시지**: `"유효하지 않은 피드백 타입입니다: {type}"`
- **원인**: 허용되지 않은 피드백 타입 사용
- **해결**: 유효한 타입 사용: `bug`, `feature`, `improvement`, `question`

#### 상태 에러
- **에러 메시지**: `"유효하지 않은 상태입니다: {status}"`
- **원인**: 허용되지 않은 상태 사용
- **해결**: 유효한 상태 사용: `open`, `in_progress`, `resolved`, `closed`

#### 이미지 개수 초과
- **에러 메시지**: `"이미지는 최대 3장까지 첨부할 수 있습니다"`
- **원인**: 피드백에 4장 이상의 이미지 첨부 시도
- **해결**: 이미지 개수 제한 준수

## 데이터 흐름

### 피드백 생성 흐름

```
1. 사용자가 피드백 작성 폼 제출
   ↓
2. 프론트엔드에서 이미지 업로드 (선택)
   ↓
3. v1_create_feedback RPC 함수 호출
   ↓
4. 인증 확인 (auth.uid())
   ↓
5. 사용자 ID 조회 (tbl_users)
   ↓
6. 데이터 검증 (타입, 이미지 개수 등)
   ↓
7. 포스트 생성 (tbl_posts INSERT)
   ↓
8. 피드백 정보 생성 (tbl_feedbacks INSERT)
   ↓
9. 포스트 ID 반환
   ↓
10. 프론트엔드에서 피드백 목록 새로고침 또는 상세 페이지로 이동
```

### 피드백 투표 흐름

```
1. 사용자가 투표 버튼 클릭
   ↓
2. v1_toggle_feedback_vote RPC 함수 호출
   ↓
3. 인증 확인 (auth.uid())
   ↓
4. 사용자 ID 조회 (tbl_users)
   ↓
5. 현재 투표 상태 확인
   ↓
6. 투표 추가 또는 제거
   ↓
7. 트리거 실행:
   - votes_count 증가 또는 감소
   ↓
8. 성공 여부 반환
   ↓
9. 프론트엔드에서 투표 수 업데이트
```

### 피드백 수정 흐름 (프로젝트 멤버)

```
1. 프로젝트 멤버가 피드백 수정 폼 제출
   ↓
2. v1_update_feedback RPC 함수 호출
   ↓
3. 인증 확인 (auth.uid())
   ↓
4. 사용자 ID 조회 (tbl_users)
   ↓
5. 프로젝트 멤버 확인
   ↓
6. 피드백 정보 수정 (tbl_feedbacks UPDATE)
   ↓
7. 포스트 정보 수정 (필요한 경우, tbl_posts UPDATE)
   ↓
8. 성공 여부 반환
   ↓
9. 프론트엔드에서 피드백 정보 업데이트
```

## 참고사항

### 데이터 형식

- **이미지 배열**: JSONB 배열 형식, 예: `["url1", "url2"]`
- **날짜/시간**: ISO 8601 형식 (timestamptz)

### 트리거 동작

- 모든 카운트(`votes_count`)는 트리거를 통해 자동 동기화
- `updated_at`은 트리거를 통해 자동 업데이트

### 소프트 삭제

- 피드백은 소프트 삭제로 처리 (`is_deleted = true`)
- 삭제된 피드백은 목록에서 제외

### 피드백 ID vs 포스트 ID

- `feedback_id`: 실제 피드백 ID (`tbl_feedbacks.id`)
- `post_id`: 포스트 ID (`tbl_posts.id`)
- 라우팅에는 `post_id` 사용
- 투표 등 피드백 전용 작업에는 `feedback_id` 사용
