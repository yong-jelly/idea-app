# 유저 아이디어

이 문서는 일반 사용자, 메이커, 학생/수료생 관점에서 우선순위가 높은 개선 아이디어를 정리합니다.

## P1

### 프로젝트 팔로우 / 구독

- 해결 문제: 프로젝트 재방문 동기 부족
- 근거: 현재 알림/구독 흐름 부재
- 제안:
  - 프로젝트 구독
  - 새 업데이트 알림
  - 주간 요약 메일

### 첫 피드백 남기기 가이드

- 해결 문제: 첫 참여 진입장벽
- 근거: 빈 상태 안내 부족
- 제안:
  - 질문 템플릿
  - 버그/기능요청 예시
  - "이 프로젝트에 가장 도움 되는 의견" 안내

### 업데이트 신뢰 배지

- 해결 문제: 살아 있는 프로젝트 구분 어려움
- 근거: 콘셉트는 실행력 중심인데 UI 근거 약함
- 제안:
  - 최근 업데이트
  - 응답률
  - 해결률
  - 최근 30일 활동성 배지

### 데모 검증 체크리스트

- 해결 문제: 사용자 입장에서 프로젝트를 어떻게 써볼지 애매함
- 근거: 상세가 소개형 중심
- 제안:
  - "이 3가지를 먼저 해보세요"
  - 주요 기능 체크리스트
  - 알려진 제한사항 안내

### 사용자 제안 보드

- 해결 문제: 요청이 댓글로 흩어짐
- 근거: 구조화된 피드백 방향은 있으나 보드 관점이 약함
- 제안:
  - 인기 요청
  - 최근 해결
  - 중복 제안 병합
  - 배포 연동

## P2

### 포트폴리오 타임라인

- 해결 문제: 학생/주니어 성장 서사 부족
- 근거: 프로필은 있지만 성장 맥락이 약함
- 제안:
  - 프로젝트 시작
  - 주요 피드백
  - 릴리즈
  - 성과를 한 타임라인으로 제공

### 프로젝트 비교 보기

- 해결 문제: 유사 프로젝트 탐색 어려움
- 근거: 탐색 고도화 TODO 존재
- 제안:
  - 카테고리
  - 스택
  - 활동성
  - 응답률 비교 카드

### 멘토 질문 모드

- 해결 문제: 피드백 품질 편차
- 근거: 교육기관/멘토 사용 시나리오 필요
- 제안:
  - 멘토 전용 질문 템플릿
  - 피드백 타입 확장
  - 리뷰 강조 UI

### 베타 테스터 모집

- 해결 문제: 초기 사용자 확보
- 근거: 리워드와 참여 보상 방향은 있으나 운영 구조 부재
- 제안:
  - 신청 폼
  - 초대 코드
  - 테스트 대상 기능 지정

### 프로젝트 공개 옵션

- 해결 문제: 모든 프로젝트가 동일한 공개 정책을 가짐
- 근거: 교육기관, 심사대기, 비공개 공유 시나리오 필요
- 제안:
  - 공개
  - 기관/기수 한정
  - 링크 공유
  - 심사대기

## 메모

- 리워드는 별도 기능 트랙으로 보류합니다.
- 유저 관점에서 당장 중요한 것은 `참여 시작`, `재방문`, `신뢰`입니다.
