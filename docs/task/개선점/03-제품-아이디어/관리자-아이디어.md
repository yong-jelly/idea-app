# 관리자 아이디어

이 문서는 프로젝트 관리자, 플랫폼 운영자, 교육기관 운영자 관점의 개선 아이디어를 정리합니다.

## P1

### 팀 권한 모델

- 해결 문제: 오너 1인 중심 구조
- 근거: 팀 테이블/멤버십 부재
- 제안:
  - `owner`
  - `editor`
  - `moderator`
  - `reviewer`
  - `mentor`

### 피드백 트리아지 보드

- 해결 문제: 피드백 처리 비체계적
- 근거: 상태/우선순위는 있으나 운영 화면 부족
- 제안:
  - `inbox`
  - `triage`
  - `in_progress`
  - `shipped`
  - 담당자, SLA, 응답 지연 표시

### 운영 대시보드

- 해결 문제: 현재 무엇이 밀렸는지 보이지 않음
- 근거: 관리자용 집계 화면 부재
- 제안:
  - 신규 피드백 수
  - 미응답 피드백
  - 최근 배포
  - 활동 사용자 수

### 신고 / 모더레이션 센터

- 해결 문제: 운영 리스크 대응 수단 부족
- 근거: 전역 운영자 기능 부재
- 제안:
  - 신고함
  - 숨김
  - 정지
  - 감사 로그

### 기관 / 코호트 관리

- 해결 문제: 학원/교육기관 운영 구조 없음
- 근거: 기관 모델 부재
- 제안:
  - 기관 프로필
  - 과정
  - 기수
  - 학생 연결
  - 전시 페이지

### 프로젝트 승인 워크플로

- 해결 문제: 검수와 공개 기준 부재
- 근거: `draft/review/approved/published` 상태 없음
- 제안:
  - 초안
  - 검수중
  - 승인됨
  - 공개됨

## P2

### 멘토 / 심사 리뷰 모듈

- 해결 문제: 교육기관용 평가 체계 부재
- 제안:
  - 기준표
  - 리뷰 기록
  - 추천작 선정

### 자동 changelog 초안

- 해결 문제: 수기 운영 부담
- 제안:
  - GitHub 릴리즈
  - 커밋/PR 요약
  - 배포 노트 초안 생성

### 코호트 피드 / 데모데이 모드

- 해결 문제: 기관 소개 UX 약함
- 제안:
  - 기수별 랜딩
  - 발표순 정렬
  - 우수작 큐레이션

## 보류

### 리워드 원장 시스템

- 상태: 보류
- 이유:
  - 리워드 정책
  - 포인트 적립 기준
  - 재고/쿠폰/환불 정책이 아직 미정
- 현재 방침:
  - 실기능 개발 보류
  - UI 노출 방식만 정리

## 메모

- 관리자 기능은 `프로젝트 운영`, `플랫폼 운영`, `교육기관 운영`으로 나눠 생각해야 합니다.
- 지금은 리워드보다 권한 모델과 운영 대시보드가 훨씬 우선입니다.
