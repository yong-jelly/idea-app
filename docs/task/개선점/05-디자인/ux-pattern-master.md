# UI/UX Pattern Master

## 정보 구조

- 1차 계층: `발견`, `프로젝트`, `커뮤니티`
- 2차 계층: 프로젝트 상세 내부의 `개요`, `업데이트`, `피드백`, `마일스톤`, `변경사항`
- 3차 계층: 운영 액션은 우측 패널 또는 상단 액션바로 분리

## 플랫폼 패턴

- 피드: SNS형 발견 경험
- 피드백: GitHub Issues형 구조화 경험
- 변경사항: 릴리즈 노트형 신뢰 경험
- 교육기관 모드: 쇼케이스형 전시 경험

## 핵심 8개 화면

### 1. 메인 피드

- 목적: 전체 서비스의 최신 움직임 전달
- 핵심 요소: 추천 프로젝트, 최근 업데이트, 최근 해결된 피드백

### 2. 탐색

- 목적: 관심 프로젝트 빠른 발견
- 핵심 요소: 카테고리, 스택, 활동성, 응답률

### 3. 프로젝트 상세

- 목적: 제품 소개와 신뢰 형성
- 핵심 요소: 소개, 데모, 저장소, 최근 개선, 팀/기관 정보

### 4. 커뮤니티 허브

- 목적: 공지, 피드백, 마일스톤, 변경사항 요약
- 핵심 요소: 미응답 피드백, 최근 배포, 이번 주 목표

### 5. 피드백 목록 / 상세

- 목적: 구조화된 제안과 처리 흐름 관리
- 핵심 요소: 상태, 우선순위, 공식 답변, 배포 연결

### 6. 마일스톤 / 공개 로드맵

- 목적: 앞으로의 계획 신뢰 있게 전달
- 핵심 요소: 현재 버전, 다음 버전, 완료율

### 7. 변경사항

- 목적: 실제 실행력 증명
- 핵심 요소: 버전, 핵심 변경점, 관련 요청

### 8. 프로필 / 포트폴리오 허브

- 목적: 개인 결과물과 성장 기록 집약
- 핵심 요소: 만든 프로젝트, 받은 피드백, 활동 이력, 기관 정보

## 컴포넌트 제안

- `ProjectTrustCard`
- `CommunityHealthPanel`
- `FeedbackLifecycle`
- `InstitutionBadge`
- `BetaFeatureLabel`

## 접근성 기준

- 색상 외에도 상태 아이콘과 텍스트 제공
- 버튼에 명확한 라벨 제공
- 로딩, 오류, 빈 상태 문구 명시
- WCAG AA 수준 대비 유지

## 마이크로 인터랙션

- 좋아요/투표 즉시 반응, 실패 시 롤백
- 변경사항 게시 후 관련 피드백 연결
- 마일스톤 완료 시 업데이트 작성 유도

## 반응형 원칙

- 모바일: 읽기와 참여 우선
- 태블릿: 컨텐츠 + 보조 패널 2단
- 데스크톱: 탐색 + 본문 + 상태 패널 3단
