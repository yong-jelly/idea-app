# 교육기관 도입

## 권장 포지셔닝

1DD는 교육과정 결과물을 올리는 단순 전시판이 아니라, 수강생과 수료생의 프로젝트가 실제 사용자와 만나고 피드백을 받고 개선되며 성장하는 실전 포트폴리오 플랫폼입니다.

## 왜 맞는가

- `프로젝트 등록 + 프로필 + 피드백 + 마일스톤 + 변경사항` 구조가 이미 있음
- 결과물만 보여주는 것이 아니라 성장 이력까지 보여줄 수 있음
- 수료생은 "무엇을 만들었는가"뿐 아니라 "어떻게 개선했는가"를 증명할 수 있음

## 핵심 시나리오

1. 기관이 기관 프로필을 개설합니다.
2. 과정과 기수 페이지를 만듭니다.
3. 학생이 프로젝트 템플릿으로 등록합니다.
4. 멘토가 공식 피드백과 리뷰를 남깁니다.
5. 검수를 거쳐 데모데이 또는 공개 피드에 노출합니다.
6. 수료 후에는 개인 프로필과 연결된 살아 있는 포트폴리오로 남깁니다.

## 필요한 엔티티

- 기관
- 과정
- 기수
- 학생 소속
- 멘토 소속
- 승인 상태
- 공개 정책

## 필요한 운영 기능

- 학생 초대와 일괄 등록
- 제출 마감
- 승인 / 반려 / 수정 요청
- 멘토 피드백 구분
- 우수작 큐레이션
- 기수별 쇼케이스 페이지

## 기대 효과

- 교육기관:
  - 결과물 나열을 넘어 실전 프로젝트 운영 역량을 보여줄 수 있음
- 학생:
  - 실제 사용자 반응과 개선 기록이 포함된 포트폴리오 확보
- 플랫폼:
  - 공개 커뮤니티와 교육기관 B2B를 동시에 연결 가능

## 주의점

- 기관 도입은 별도 권한 모델 없이 시작하면 운영이 금방 꼬입니다.
- 학생용 포트폴리오와 기관용 검수 워크플로를 같이 설계해야 합니다.
