# 서비스 이해

## 서비스 한 줄 정의

1DD는 빠르게 만든 초기 제품을 공개하고, 사용자 피드백과 업데이트 이력을 통해 다음 버전으로 성장시키는 공개 개발 플랫폼입니다.

## 현재 제품 성격

- 단순 포트폴리오 전시 사이트보다 `공개 운영형 프로젝트 플랫폼`에 가깝습니다.
- 현재 구현의 핵심 축은 `프로젝트 등록`, `메인 피드`, `프로젝트 상세`, `프로젝트별 커뮤니티`, `피드백`, `마일스톤`, `변경사항`입니다.
- 따라서 이 서비스의 중심 가치는 "무엇을 만들었는가"보다 "사용자와 어떻게 개선하고 있는가"에 있습니다.

## 핵심 사용자

| 사용자 | 주요 목표 | 현재 맞는 정도 | 현재 공백 |
|---|---|---:|---|
| 1인 메이커 | MVP 공개, 첫 반응 확보 | 높음 | 운영 도구, 팀 기능 부족 |
| 초급 개발자/비개발 메이커 | 포트폴리오와 실사용 검증 동시 확보 | 중간 | 온보딩, 가이드 부족 |
| 얼리어답터 사용자 | 새 프로젝트 발견, 피드백 참여 | 중간 | 구독/알림/리워드 미완 |
| 프로젝트 관리자 | 피드백, 공지, 변경사항 관리 | 중간 | 권한/대시보드/트리아지 부족 |
| 교육기관 담당자 | 과정/기수 단위 운영 | 낮음 | 기관 모델 부재 |
| 수강생/수료생 | 프로젝트를 실전형 포트폴리오로 만들기 | 중간 | 기관용 제출/검수 흐름 부족 |

## 운영자 관점

| 운영 주체 | 필요한 기능 |
|---|---|
| 프로젝트 오너 | 공지 작성, 피드백 상태 관리, 마일스톤, 변경사항 |
| 프로젝트 팀 멤버 | 공동 운영, 담당자 배정, 응답 작성 |
| 플랫폼 운영자 | 검수, 신고 처리, 사용자/프로젝트 제재 |
| 교육기관 운영자 | 기관/과정/기수 관리, 학생 승인, 데모데이 큐레이션 |
| 멘토/심사위원 | 리뷰, 멘토 피드백, 추천작 선정 |

## 핵심 사용자 플로우

1. 메이커가 프로젝트를 등록합니다.
2. 프로젝트가 메인 피드와 탐색 화면에 노출됩니다.
3. 사용자가 프로젝트 상세에서 프로젝트를 이해합니다.
4. 사용자가 커뮤니티에서 공지, 피드백, 마일스톤, 변경사항을 확인합니다.
5. 사용자가 댓글, 투표, 버그 제보, 기능 요청으로 참여합니다.
6. 프로젝트 관리자가 상태를 변경하고 공식 답변과 업데이트를 남깁니다.
7. 프로젝트는 지속적으로 움직이는 운영 타임라인이 됩니다.

## 현재 구조 요약

- 라우트 중심 파일: `src/app/router/index.tsx`
- 주요 라우트:
  - `/`
  - `/explore`
  - `/create-project`
  - `/project/:id`
  - `/project/:id/community`
  - `/profile/:username`
- 주요 데이터 엔티티:
  - `odd.tbl_users`
  - `odd.projects`
  - `odd.tbl_posts`
  - `odd.tbl_feedbacks`
  - `odd.tbl_comments`
  - `odd.tbl_milestones`
  - `odd.tbl_tasks`
  - `odd.tbl_changelogs`

## 핵심 판단

1DD는 현재도 아이디어 검증 사이트가 아니라, 잘 다듬으면 `공개 제품 운영 + 실전 포트폴리오 + 교육기관 쇼케이스`를 묶는 플랫폼으로 갈 수 있습니다.
