# 커뮤니티 공지 시스템

## 개요

프로젝트 커뮤니티에서 개발팀이 공지사항, 업데이트, 투표를 작성하고 관리하는 시스템입니다. 프로젝트 생성자만 공지를 작성/수정/삭제할 수 있으며, 일반 사용자는 조회 및 댓글 작성만 가능합니다.

### 주요 기능

- **포스트 작성/수정/삭제**: 공지사항, 업데이트, 투표 포스트 관리
- **상단 고정 기능**: 중요한 공지를 상단에 고정
- **이미지 첨부**: 최대 3개 이미지 첨부 가능
- **투표 기능**: 투표 타입 포스트에서 최소 2개, 최대 5개 옵션 제공
- **댓글 시스템**: 포스트별 댓글 작성/수정/삭제 (최대 3단계 깊이)
- **필터링**: 전체/공지/업데이트/투표 타입별 필터링
- **페이지네이션**: 30개씩 페이지네이션 지원

## 데이터베이스 스키마

### 1. 공지사항/업데이트/투표 포스트 파생 테이블 (`odd.tbl_post_announcements`)

공지사항, 업데이트, 투표 포스트의 추가 정보를 저장하는 테이블입니다. `tbl_posts`와 1:1 관계입니다.

#### 스키마

```sql
CREATE TABLE odd.tbl_post_announcements (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    post_id uuid NOT NULL REFERENCES odd.tbl_posts(id) ON DELETE CASCADE,
    
    -- 포스트 정보
    title text NOT NULL,  -- 제목
    post_type text NOT NULL CHECK (post_type IN ('announcement', 'update', 'vote')),  -- 포스트 타입
    
    -- 상태
    is_pinned boolean DEFAULT false NOT NULL,
    
    -- 타임스탬프
    created_at timestamptz DEFAULT now() NOT NULL,
    updated_at timestamptz DEFAULT now() NOT NULL,
    
    -- 하나의 포스트는 하나의 announcement만 가질 수 있음
    UNIQUE(post_id)
);
```

#### 주요 컬럼 설명

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `id` | uuid | ✅ | 고유 ID (자동 생성) |
| `post_id` | uuid | ✅ | 포스트 ID (`odd.tbl_posts.id` 참조) |
| `title` | text | ✅ | 제목 |
| `post_type` | text | ✅ | 포스트 타입: `announcement`, `update`, `vote` |
| `is_pinned` | boolean | ✅ | 상단 고정 여부 |
| `created_at` | timestamptz | ✅ | 생성일시 |
| `updated_at` | timestamptz | ✅ | 수정일시 (트리거로 자동 업데이트) |

#### 인덱스

| 인덱스명 | 컬럼 | 용도 |
|---------|------|------|
| `idx_tbl_post_announcements_post_id` | `post_id` | 포스트 ID로 조회 |
| `idx_tbl_post_announcements_type` | `post_type` | 포스트 타입별 조회 |
| `idx_tbl_post_announcements_pinned` | `is_pinned` (WHERE `is_pinned = true`) | 고정된 포스트 조회 |

### 2. 투표 옵션 테이블 (`odd.tbl_post_votes`)

투표 포스트의 옵션을 저장하는 테이블입니다. 하나의 투표 포스트는 여러 개의 옵션을 가질 수 있습니다.

#### 스키마

```sql
CREATE TABLE odd.tbl_post_votes (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    post_id uuid NOT NULL REFERENCES odd.tbl_posts(id) ON DELETE CASCADE,
    
    -- 옵션 정보
    option_text text NOT NULL,  -- 옵션 텍스트 (최대 50자)
    sort_order integer NOT NULL DEFAULT 0,  -- 정렬 순서
    
    -- 통계 (비정규화)
    votes_count integer DEFAULT 0 NOT NULL,
    
    -- 타임스탬프
    created_at timestamptz DEFAULT now() NOT NULL,
    updated_at timestamptz DEFAULT now() NOT NULL
);
```

#### 주요 컬럼 설명

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `id` | uuid | ✅ | 고유 ID (자동 생성) |
| `post_id` | uuid | ✅ | 포스트 ID (`odd.tbl_posts.id` 참조) |
| `option_text` | text | ✅ | 옵션 텍스트 |
| `sort_order` | integer | ✅ | 정렬 순서 (0부터 시작) |
| `votes_count` | integer | ✅ | 투표 수 (비정규화, 트리거로 자동 동기화) |
| `created_at` | timestamptz | ✅ | 생성일시 |
| `updated_at` | timestamptz | ✅ | 수정일시 (트리거로 자동 업데이트) |

#### 인덱스

| 인덱스명 | 컬럼 | 용도 |
|---------|------|------|
| `idx_tbl_post_votes_post_id` | `post_id, sort_order` | 포스트별 투표 옵션 조회 (정렬 순서 기준) |

### 3. 투표 응답 테이블 (`odd.tbl_post_vote_responses`)

사용자의 투표 응답을 저장하는 테이블입니다. 한 사용자는 한 포스트에 하나의 투표만 가능합니다.

#### 스키마

```sql
CREATE TABLE odd.tbl_post_vote_responses (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    post_id uuid NOT NULL REFERENCES odd.tbl_posts(id) ON DELETE CASCADE,
    vote_option_id uuid NOT NULL REFERENCES odd.tbl_post_votes(id) ON DELETE CASCADE,
    user_id bigint NOT NULL REFERENCES odd.tbl_users(id) ON DELETE CASCADE,
    
    created_at timestamptz DEFAULT now() NOT NULL,
    
    -- 한 사용자는 한 포스트에 하나의 투표만 가능
    UNIQUE(post_id, user_id)
);
```

#### 주요 컬럼 설명

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `id` | uuid | ✅ | 고유 ID (자동 생성) |
| `post_id` | uuid | ✅ | 포스트 ID (`odd.tbl_posts.id` 참조) |
| `vote_option_id` | uuid | ✅ | 투표 옵션 ID (`odd.tbl_post_votes.id` 참조) |
| `user_id` | bigint | ✅ | 투표한 사용자 ID (`odd.tbl_users.id` 참조) |
| `created_at` | timestamptz | ✅ | 생성일시 |

#### 인덱스

| 인덱스명 | 컬럼 | 용도 |
|---------|------|------|
| `idx_tbl_post_vote_responses_post_id` | `post_id` | 포스트별 투표 응답 조회 |
| `idx_tbl_post_vote_responses_option_id` | `vote_option_id` | 옵션별 투표 응답 조회 |
| `idx_tbl_post_vote_responses_user_id` | `user_id` | 사용자별 투표 조회 |

#### 트리거

- **`trigger_increment_post_vote_option_count`**: 투표 응답 추가 시 옵션의 `votes_count` 증가
- **`trigger_decrement_post_vote_option_count`**: 투표 응답 삭제 시 옵션의 `votes_count` 감소

## RPC 함수

### 1. 커뮤니티 포스트 조회 (`odd.v1_fetch_community_posts`)

커뮤니티 포스트 목록을 조회하는 함수입니다. 공지사항, 업데이트, 투표 포스트를 포함하며 투표 정보도 함께 반환합니다.

#### 함수 시그니처

```sql
CREATE OR REPLACE FUNCTION odd.v1_fetch_community_posts(
    p_project_id uuid,
    p_post_type text DEFAULT NULL,
    p_limit integer DEFAULT 30,
    p_offset integer DEFAULT 0
)
RETURNS TABLE (
    -- 포스트 기본 정보
    id uuid,
    author_id bigint,
    type text,
    content text,
    images jsonb,
    likes_count integer,
    comments_count integer,
    is_pinned boolean,
    created_at timestamptz,
    updated_at timestamptz,
    -- 작성자 정보
    author_username text,
    author_display_name text,
    author_avatar_url text,
    -- 현재 사용자 인터랙션 상태
    is_liked boolean,
    -- 공지사항 정보
    title text,
    post_type text,
    -- 투표 정보 (투표 타입일 때만)
    vote_options jsonb,
    voted_option_id uuid,
    total_votes integer
)
```

#### 매개변수

| 매개변수 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| `p_project_id` | uuid | ✅ | 프로젝트 ID |
| `p_post_type` | text | - | 포스트 타입 필터: `announcement`, `update`, `vote` (NULL이면 모든 타입) |
| `p_limit` | integer | - | 조회 개수 제한 (기본값: 30, 최대: 100) |
| `p_offset` | integer | - | 페이지네이션 오프셋 (기본값: 0) |

#### 반환 데이터 구조

```typescript
interface CommunityPost {
  // 포스트 기본 정보
  id: string;
  author_id: number;
  type: string;
  content: string;
  images: string[];
  likes_count: number;
  comments_count: number;
  is_pinned: boolean;
  created_at: string;
  updated_at: string;
  
  // 작성자 정보
  author_username: string;
  author_display_name: string;
  author_avatar_url?: string;
  
  // 현재 사용자 인터랙션 상태
  is_liked: boolean;
  
  // 공지사항 정보
  title: string;
  post_type: 'announcement' | 'update' | 'vote';
  
  // 투표 정보 (투표 타입일 때만)
  vote_options?: Array<{
    id: string;
    text: string;
    votesCount: number;
    sortOrder: number;
  }>;
  voted_option_id?: string;
  total_votes?: number;
}
```

#### 사용 예시

```typescript
import { supabase } from "@/shared/lib/supabase";

// 전체 포스트 조회
const { data, error } = await supabase
  .schema('odd')
  .rpc('v1_fetch_community_posts', {
    p_project_id: 'project-uuid-here',
    p_limit: 30,
    p_offset: 0
  });

// 공지사항만 조회
const { data, error } = await supabase
  .schema('odd')
  .rpc('v1_fetch_community_posts', {
    p_project_id: 'project-uuid-here',
    p_post_type: 'announcement',
    p_limit: 30,
    p_offset: 0
  });
```

#### 정렬 규칙

1. 고정된 포스트를 먼저 표시 (`is_pinned DESC`)
2. 최신순 정렬 (`created_at DESC`)

### 2. 커뮤니티 포스트 생성 (`odd.v1_create_community_post`)

커뮤니티 포스트를 생성하는 함수입니다. 포스트와 공지사항 정보를 함께 생성하며, 투표 타입일 경우 투표 옵션도 함께 생성합니다.

#### 함수 시그니처

```sql
CREATE OR REPLACE FUNCTION odd.v1_create_community_post(
    p_project_id uuid,
    p_post_type text,  -- 'announcement' | 'update' | 'vote'
    p_title text,
    p_content text,
    p_images jsonb DEFAULT '[]'::jsonb,
    p_is_pinned boolean DEFAULT false,
    p_vote_options text[] DEFAULT NULL  -- 투표 타입일 때만 사용 (최소 2개, 최대 5개)
)
RETURNS uuid
```

#### 매개변수

| 매개변수 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| `p_project_id` | uuid | ✅ | 프로젝트 ID |
| `p_post_type` | text | ✅ | 포스트 타입: `announcement`, `update`, `vote` |
| `p_title` | text | ✅ | 제목 |
| `p_content` | text | ✅ | 내용 |
| `p_images` | jsonb | - | 이미지 URL 배열 (최대 3개) |
| `p_is_pinned` | boolean | - | 상단 고정 여부 (기본값: false) |
| `p_vote_options` | text[] | - | 투표 옵션 배열 (투표 타입일 때만, 최소 2개, 최대 5개) |

#### 검증 규칙

- 포스트 타입이 유효한지 확인 (`announcement`, `update`, `vote`)
- 투표 타입일 경우 옵션이 최소 2개, 최대 5개인지 확인
- 이미지 개수가 3개 이하인지 확인
- 프로젝트 작성자만 포스트 생성 가능

#### 사용 예시

```typescript
import { supabase } from "@/shared/lib/supabase";

// 공지사항 생성
const { data, error } = await supabase
  .schema('odd')
  .rpc('v1_create_community_post', {
    p_project_id: 'project-uuid-here',
    p_post_type: 'announcement',
    p_title: '새로운 기능 출시 안내',
    p_content: '다음 주에 새로운 기능이 출시됩니다.',
    p_images: ['url1', 'url2'],
    p_is_pinned: true
  });

// 투표 포스트 생성
const { data, error } = await supabase
  .schema('odd')
  .rpc('v1_create_community_post', {
    p_project_id: 'project-uuid-here',
    p_post_type: 'vote',
    p_title: '다음 기능 우선순위 투표',
    p_content: '다음에 구현할 기능을 선택해주세요.',
    p_vote_options: ['다크모드', '다국어 지원', '알림 기능', '검색 기능']
  });
```

## 프론트엔드 사용 방법

### 1. 포스트 목록 조회

```typescript
import { supabase } from "@/shared/lib/supabase";

async function fetchCommunityPosts(projectId: string, postType?: string, limit = 30, offset = 0) {
  const { data, error } = await supabase
    .schema('odd')
    .rpc('v1_fetch_community_posts', {
      p_project_id: projectId,
      p_post_type: postType || null,
      p_limit: limit,
      p_offset: offset
    });

  if (error) {
    console.error('포스트 조회 실패:', error);
    return { error };
  }

  return { data };
}
```

### 2. 포스트 생성

```typescript
import { supabase } from "@/shared/lib/supabase";
import { uploadPostImages } from "@/shared/lib/storage";

async function createCommunityPost(postData: {
  projectId: string;
  postType: 'announcement' | 'update' | 'vote';
  title: string;
  content: string;
  images?: File[];
  isPinned?: boolean;
  voteOptions?: string[];
}) {
  // 이미지 업로드
  let imagePaths: string[] = [];
  if (postData.images && postData.images.length > 0) {
    const { data: { user } } = await supabase.auth.getUser();
    if (!user) {
      return { error: new Error('로그인이 필요합니다') };
    }

    const tempPostId = `temp-${Date.now()}`;
    const { paths, error: uploadError } = await uploadPostImages(
      postData.images,
      user.id,
      tempPostId
    );

    if (uploadError) {
      return { error: uploadError };
    }

    imagePaths = paths;
  }

  // 포스트 생성
  const { data, error } = await supabase
    .schema('odd')
    .rpc('v1_create_community_post', {
      p_project_id: postData.projectId,
      p_post_type: postData.postType,
      p_title: postData.title,
      p_content: postData.content,
      p_images: imagePaths,
      p_is_pinned: postData.isPinned || false,
      p_vote_options: postData.voteOptions || null
    });

  if (error) {
    console.error('포스트 생성 실패:', error);
```

## 보안 및 권한 (RLS)

### 포스트 테이블 (`odd.tbl_posts`)

#### 읽기 권한
- 모든 사용자(공개)가 삭제되지 않은 포스트를 읽을 수 있음
- 정책: `"Anyone can read active posts"` (SELECT, TO public, USING (`is_deleted = false`))

#### 쓰기 권한

**포스트 생성 (INSERT):**
- 프로젝트 작성자만 포스트 생성 가능
- 정책: `"Authenticated users can create posts"` (INSERT, TO authenticated)
- 함수 내에서 프로젝트 작성자 확인

**포스트 수정 (UPDATE):**
- 작성자만 자신의 포스트를 수정할 수 있음
- 정책: `"Users can update own posts"` (UPDATE, TO authenticated)

**포스트 삭제 (DELETE):**
- 작성자만 자신의 포스트를 삭제할 수 있음
- 정책: `"Users can delete own posts"` (DELETE, TO authenticated)

### 공지사항 테이블 (`odd.tbl_post_announcements`)

#### 읽기 권한
- 모든 사용자가 읽을 수 있음
- 정책: `"Anyone can read post announcements"` (SELECT, TO public)

#### 쓰기 권한
- 인증된 사용자가 생성 가능 (권한 체크는 함수에서 수행)
- 작성자만 수정 가능 (권한 체크는 함수에서 수행)

### 투표 응답 테이블 (`odd.tbl_post_vote_responses`)

#### 읽기 권한
- 모든 사용자가 읽을 수 있음
- 정책: `"Anyone can read post vote responses"` (SELECT, TO public)

#### 쓰기 권한
- 인증된 사용자만 투표 응답 생성 가능
- 자신의 투표만 삭제 가능

## 에러 처리

### 일반적인 에러

#### 인증 에러
- **에러 메시지**: `"인증이 필요합니다"`
- **원인**: 로그인하지 않은 사용자가 인증이 필요한 작업을 시도
- **해결**: 로그인 페이지로 리다이렉트

#### 권한 에러
- **에러 메시지**: `"프로젝트 작성자만 포스트를 작성할 수 있습니다"`
- **원인**: 프로젝트 작성자가 아닌 사용자가 포스트 작성 시도
- **해결**: 권한 확인 후 작업 수행

### 포스트 관련 에러

#### 포스트 타입 에러
- **에러 메시지**: `"유효하지 않은 포스트 타입입니다: {type}"`
- **원인**: 허용되지 않은 포스트 타입 사용
- **해결**: 유효한 타입 사용: `announcement`, `update`, `vote`

#### 이미지 개수 초과
- **에러 메시지**: `"이미지는 최대 3장까지 첨부할 수 있습니다"`
- **원인**: 포스트에 4장 이상의 이미지 첨부 시도
- **해결**: 이미지 개수 제한 준수

#### 투표 옵션 에러
- **에러 메시지**: `"투표 옵션은 최소 2개, 최대 5개까지 가능합니다"`
- **원인**: 투표 타입 포스트에 옵션이 1개 이하 또는 6개 이상
- **해결**: 옵션 개수 제한 준수

## 데이터 흐름

### 포스트 생성 흐름

```
1. 사용자가 포스트 작성 폼 제출
   ↓
2. 프론트엔드에서 이미지 업로드 (선택)
   ↓
3. v1_create_community_post RPC 함수 호출
   ↓
4. 인증 확인 (auth.uid())
   ↓
5. 사용자 ID 조회 (tbl_users)
   ↓
6. 프로젝트 작성자 확인
   ↓
7. 데이터 검증 (타입, 이미지 개수, 투표 옵션 등)
   ↓
8. 포스트 생성 (tbl_posts INSERT)
   ↓
9. 공지사항 정보 생성 (tbl_post_announcements INSERT)
   ↓
10. 투표 옵션 생성 (투표 타입일 경우, tbl_post_votes INSERT)
   ↓
11. 포스트 ID 반환
   ↓
12. 프론트엔드에서 포스트 목록 새로고침
```

### 투표 응답 생성 흐름

```
1. 사용자가 투표 옵션 선택
   ↓
2. v1_create_vote_response RPC 함수 호출
   ↓
3. 인증 확인 (auth.uid())
   ↓
4. 사용자 ID 조회 (tbl_users)
   ↓
5. 기존 투표 확인
   ↓
6. 기존 투표가 있으면 삭제 후 새 투표 추가
   ↓
7. 투표 응답 생성 (tbl_post_vote_responses INSERT)
   ↓
8. 트리거 실행:
   - 옵션의 votes_count 증가
   ↓
9. 성공 여부 반환
   ↓
10. 프론트엔드에서 투표 결과 업데이트
```

## 참고사항

### 데이터 형식

- **이미지 배열**: JSONB 배열 형식, 예: `["url1", "url2"]`
